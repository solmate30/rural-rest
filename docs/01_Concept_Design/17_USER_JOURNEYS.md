# 17. 유저 여정 (User Journeys)

> Created: 2026-04-07 00:00
> Updated: 2026-04-07 00:00

Rural Rest의 4가지 유저 타입별 핵심 여정을 정의한다.

---

## 1. 여행자 여정 — "빈집에서의 하룻밤"

**페르소나**: 다현 (28세, 프리랜서 디자이너, 서울)
**목표**: 주말에 서울을 벗어나 조용한 시골집에서 혼자만의 시간을 갖고 싶다.

### 전체 흐름

```
홈 진입 ──▶ 숙소 탐색 ──▶ 숙소 상세 ──▶ 예약 ──▶ 결제 ──▶ 체크인
   /            /search       /property/:id    /book/:id
```

### 단계별 상세

**Step 1. 발견** `/`
- 홈에서 지역 배지(경주, 남해 등) 클릭 또는 검색창에 지역 입력
- active 상태인 숙소만 카드로 표시됨 (펀딩 중인 숙소는 노출 안 됨)

**Step 2. 탐색** `/search`
- 지역 + 최대 금액 필터 적용
- 결과 카드에서 숙소 선택

**Step 3. 숙소 상세** `/property/:id`
- 갤러리, 편의시설, 호스트 정보, 위치 지도 확인
- 날짜 캘린더에서 체크인/아웃 선택
- **숙소 상태별 분기**:
  - `active` → 예약 버튼 활성화
  - `funding` / `funded` → 예약 버튼 없음, 투자 유도 카드 표시
  - `failed` → "환불 완료" 배지

**Step 4. 예약** `/book/:id`
- 로그인 필요 (미로그인 시 `/auth` 리다이렉트)
- 날짜, 인원 확정
- 결제 수단 선택: 카드(PayPal) 또는 USDC(Solana)
  - USDC 결제 시 KYC + 지갑 등록 필요

**Step 5. 결제 완료** `/book/success`
- 예약 ID 발급, 상태: `pending`
- 어드민 또는 마을운영자가 승인 후 `confirmed`로 전환

**Step 6. 체크인**
- 승인 후 디지털 키(QR) 발송
- 현장 도어록에 스캔하여 입실

## 2. 투자자 여정 — "빈집 재생에 투자하다"

**페르소나**: 준혁 (35세, IT 직장인, 서울)
**목표**: 소액으로 지역 재생 프로젝트에 투자하고 월 배당을 받고 싶다.

### 전체 흐름

```
투자 목록 ──▶ 물건 상세 ──▶ KYC + 지갑 ──▶ 토큰 구매 ──▶ 펀딩 확인
  /invest      /invest/:id     /kyc            (온체인)
                                    ↓
                         배당 수령 ◀── 월 정산 ◀── 숙소 오픈 후 운영
                        /my-investments
                                    ↓
                              DAO 투표
                             /governance
```

### 단계별 상세

**Step 1. 탐색** `/invest`
- 지역, 상태(펀딩 중/운영 중), APY, 테마별 필터
- 펀딩 진행률 바, 예상 연수익률 확인

**Step 2. 물건 상세** `/invest/:id`
- 토큰 정보 (mint 주소, 총 발행량, 토큰 가격, APY)
- 월 예상 수익 차트
- 펀딩 진행률, 마감 카운트다운

**Step 3. KYC + 지갑 등록**
- `/kyc` 본인 인증 (현재 시뮬레이션 3초)
- Privy 임베디드 지갑 자동 생성 (별도 설치 불필요)

**Step 4. 토큰 구매** (온체인)
- USDC 입력 → Privy 서명 팝업 → Anchor 프로그램 실행
- 에스크로 PDA에 USDC 락업, 토큰 수령
- 1인 최대 30% 보유 제한

**Step 5. 펀딩 결과 확인**
- 성공 (`active`) → 에스크로 → 운영 자금 전환, 배당 시작
- 실패 (`failed`) → `/invest/:id`에서 환불 버튼 활성화

**Step 6. 배당 수령** `/my-investments`
- 포트폴리오 요약 (투자액, 현재가치, 수익률)
- 미수령 배당 표시 → Claim 버튼 → 온체인 서명 → USDC 수령

**Step 7. DAO 투표** `/governance`
- 운영 관련 제안 (가격 정책, 운영자 변경 등) 조회
- 보유 토큰 수만큼 투표권 행사 (투표 영향력 10% 상한)

## 3. 어드민 여정 — "루랄레스트 운영자"

**페르소나**: 루랄레스트 운영팀
**목표**: 신규 숙소를 플랫폼에 등록하고, 투자 유치 후 운영을 시작한다.

### 전체 흐름

```
숙소 등록 ──▶ RWA 발행 ──▶ 펀딩 모니터링 ──▶ 자금 해제 ──▶ 예약 승인 ──▶ 월 정산
  /admin      /admin/tokenize/:id    /admin          /admin     /admin    /admin/settlements/:id
```

### 단계별 상세

**Step 1. 숙소 등록** `/admin`
- 숙소 기본 정보 + 감정가(KRW) 설정

**Step 2. 마을운영자 계정 생성**
- 어드민 UI에서 이름 + 이메일 입력 → Privy importUser → 임베디드 지갑 자동 발급
- 운영자는 이메일 OTP로 첫 로그인

**Step 3. RWA 발행** `/admin/tokenize/:id`
- 펀딩 파라미터 설정: 최소 펀딩 목표%, 마감 기한, 예상 APY
- `[토큰 발행]` → 온체인 SPL Mint 생성 → DB 저장
- 상태: `draft` → `funding`

**Step 4. 펀딩 모니터링** `/admin`
- 진행률 바, 투자자 수, 마감까지 남은 시간
- 목표 달성 시 `[Force Fund]` 버튼 (마감 전 수동 전환 가능)

**Step 5. 자금 해제** `/admin/tokenize/:id`
- `funded` 상태에서 `[Release Funds & Activate]` 클릭
- 에스크로 USDC 해제 → 숙소 `active` 전환
- 이 시점부터 여행자 예약 가능

**Step 6. 예약 승인** `/admin`
- 대기 중인 예약 목록 확인
- 카드/USDC 결제 여부 표시
- `[승인]` / `[거절]` 버튼

**Step 7. 월 정산** `/admin/settlements/:id`
- 매월 말 미정산 숙소 목록 확인
- 운영비 입력 → dryRun 미리보기 (지자체/운영자/투자자 분배액)
- `[정산 확정]` → 3자 자동 push + 투자자 vault 적립

## 4. 마을운영자 여정 — "현장을 지키는 사람"

**페르소나**: 철수 (45세, 경주 주민, 마을운영자)
**목표**: 담당 숙소의 예약을 관리하고, 게스트를 맞이하고, 매달 정산을 받는다.

### 전체 흐름

```
계정 활성화 ──▶ 대시보드 ──▶ 예약 승인 ──▶ 게스트 체크인 ──▶ 정산 확인
                 /operator     /operator      (현장)          /operator
```

### 단계별 상세

**Step 1. 계정 활성화**
- 어드민이 이름 + 이메일로 계정 생성
- 이메일로 OTP 수신 → 첫 로그인
- Privy 임베디드 지갑 자동 발급 (정산 수령용)

**Step 2. 대시보드** `/operator`
- 담당 숙소의 예약 현황 요약
- 대기 중 예약 수, 이번 달 매출, 정산 내역

**Step 3. 예약 승인** `/operator`
- 대기 중인 예약 목록 확인
- 게스트 정보, 날짜, 결제 방식 확인
- `[승인]` / `[거절]`

**Step 4. 게스트 체크인** (현장)
- 승인 완료 → 게스트에게 디지털 키(QR) 자동 발송
- 현장 도어록 스캔으로 비대면 입실

**Step 5. 정산 확인** `/operator`
- 월별 정산 내역 (매출, 운영비, 내 몫 30%)
- 정산 완료 시 지갑으로 USDC 자동 수령

---

## 5. 여정 간 연결 관계

```
어드민 (숙소 등록 + RWA 발행)
        ↓
투자자 (토큰 구매 → 펀딩 성공)
        ↓
어드민 (자금 해제 → 숙소 active)
        ↓
여행자 (예약 → 결제)
        ↓
마을운영자 (예약 승인 → 체크인 관리)
        ↓
어드민 (월 정산 실행)
        ↓
투자자 (배당 claim) / 마을운영자 (USDC 수령) / 지자체 (USDC 수령)
```
