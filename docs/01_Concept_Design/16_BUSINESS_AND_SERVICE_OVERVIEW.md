# 16. 비즈니스 구조 & 서비스 플로우 개요

> Created: 2026-04-07 00:00
> Updated: 2026-04-07 00:00

Rural Rest의 비즈니스 구조와 핵심 서비스 플로우를 정의한다.

---

## 1. 비즈니스 구조

### 참여자 관계도

```
지자체 (시/군)
  ├── 빈집 무상임대 제공
  └── 리모델링 비용 일부 지원
        ↓
루랄레스트 (플랫폼 운영사)
  ├── 사업권 취득 및 플랫폼 총괄
  ├── 마을운영자 계약 및 관리
  ├── RWA 토큰 발행 (크라우드펀딩)
  └── 투자자 모집 및 수익 분배
        ↓
마을운영자 (현장 운영자)
  ├── 루랄레스트와 계약
  ├── 숙소 현장 운영 (청소, 체크인, 유지보수)
  └── 게스트 응대
        ↓
숙소 운영 → 영업이익 발생
        ↓
3자 분배
  ├── 지자체     40%  (고정 지갑으로 자동 push)
  ├── 마을운영자  30%  (운영자 지갑으로 자동 push)
  └── RWA 투자자  30%  (vault에서 투자자 직접 claim)
```

### 참여자별 역할 요약

| 참여자 | 역할 | 수익 구조 | 플랫폼 내 role |
|--------|------|-----------|---------------|
| 지자체 | 빈집 제공, 지역 재생 | 영업이익 40% | 외부 (지갑만 등록) |
| 루랄레스트 | 플랫폼 운영, RWA 발행, 정산 | 플랫폼 수수료 | `admin` |
| 마을운영자 | 현장 운영, 예약 관리 | 영업이익 30% | `operator` |
| RWA 투자자 | 토큰 구매, 배당 수령, DAO 투표 | 영업이익 30% (지분 비례) | `guest` (+ KYC) |
| 여행자 | 숙소 탐색, 예약, 체크인 | 없음 (소비자) | `guest` |

## 2. 핵심 서비스 플로우 (5단계)

```
1. 어드민 → 숙소 등록 + RWA 토큰 발행
        ↓
2. 투자자 → 펀딩 참여 (USDC로 토큰 구매)
        ↓
   ┌── 3A. 펀딩 성공 (목표 달성 + 기한 만료)
   │         → 자금 해제 → 숙소 오픈
   │         → 배당 시작
   │
   └── 3B. 펀딩 실패 (기한 만료 + 목표 미달)
             → 투자금 전액 환불
        ↓ (3A 경로)
4. 여행자 → 숙소 예약 → 결제 → 체크인
        ↓
5. 월 정산 → 영업이익 3자 분배
             지자체 40% / 마을운영자 30% / 투자자 30%
```

---

## 3. 숙소 상태 머신

```
draft ──(Initialize)──▶ funding ──(기한 만료 + 목표 달성)──▶ funded ──(ReleaseFunds)──▶ active
                                  └──(기한 만료 + 목표 미달)──▶ failed
```

| 상태 | 의미 | 여행자 예약 | 투자자 구매 |
|------|------|------------|------------|
| `draft` | 토큰 미발행 | 불가 | 불가 |
| `funding` | 펀딩 진행 중 | 불가 (탐색만 가능) | 가능 |
| `funded` | 목표 달성, 자금 해제 대기 | 불가 | 불가 |
| `active` | 운영 중 | 가능 | 불가 (펀딩 종료) |
| `failed` | 펀딩 실패 | 불가 | 불가 (환불 가능) |

## 4. 수익 분배 상세

### 정산 공식

```
숙박 매출 (bookings 합계)
  - 운영비 (어드민 입력)
  = 영업이익 (operatingProfit)
      ├── × 0.40 → 지자체  (USDC push → VITE_LOCAL_GOV_WALLET)
      ├── × 0.30 → 마을운영자 (USDC push → operator.walletAddress)
      └── × 0.30 → 투자자 (USDC vault → 투자자 지분 비례 claim)
```

### 분배 방식

- **지자체 / 마을운영자**: 어드민이 정산 확정 시 자동 push (단일 수신자, 즉시 전달)
- **투자자**: vault에 누적, 투자자가 직접 claim (다수 수신자, 트러스트리스)

> 상세: `docs/04_Logic_Progress/13_SETTLEMENT_ARCHITECTURE.md`

## 5. RWA 토큰 구조

- **1 숙소 = 1 SPL Mint** (고정)
- **총 발행량**: 1억 토큰 (100,000,000, decimals=0)
- **토큰 가격**: 숙소 감정가(KRW) ÷ 총 발행량 (Pyth USDC/KRW 환율 적용)
- **최소 펀딩 목표**: 감정가의 60% (기본값, 어드민 조정 가능)
- **최대 보유 한도**: 1인 30% (개인 독점 방지)
- **DAO 투표권**: 토큰 1개 = 투표권 1개 (단, 투표 영향력 10% 상한)

> 상세: `docs/01_Concept_Design/10_RWA_ISSUANCE_PLAN.md`

## 6. 플랫폼 화면 구조 (역할별)

> 상세 유저 여정: [17_USER_JOURNEYS.md](./17_USER_JOURNEYS.md)

### 어드민 (`admin` role)

| 화면 | URL | 주요 기능 |
|------|-----|----------|
| 대시보드 | `/admin` | 전체 매물 현황, 예약 승인, 펀딩 모니터링 |
| 토큰 발행 | `/admin/tokenize/:id` | RWA 파라미터 설정, 온체인 발행 |
| 정산 관리 | `/admin/settlements/:id` | 월 영업이익 3자 분배 실행 |

### 마을운영자 (`operator` role)

| 화면 | URL | 주요 기능 |
|------|-----|----------|
| 대시보드 | `/operator` | 예약 확인/승인, 체크인 관리, 정산 내역 |

### 투자자 (`guest` role + KYC)

| 화면 | URL | 주요 기능 |
|------|-----|----------|
| 투자 목록 | `/invest` | RWA 매물 탐색, 필터링 |
| 투자 상세 | `/invest/:id` | 토큰 구매, 펀딩 현황 |
| 내 투자 | `/my-investments` | 포트폴리오, 배당 claim |
| 거버넌스 | `/governance` | DAO 제안 조회 및 투표 |

### 여행자 (`guest` role)

| 화면 | URL | 주요 기능 |
|------|-----|----------|
| 홈 | `/` | 매물 탐색, 스마트 검색 |
| 검색 | `/search` | 지역/가격 필터 |
| 숙소 상세 | `/property/:id` | 상세 정보, 예약 캘린더 |
| 예약 | `/book/:id` | 날짜 선택, 카드/USDC 결제 |
| 내 예약 | `/my-bookings` | 예약 이력 |
